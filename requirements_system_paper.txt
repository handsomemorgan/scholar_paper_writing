请撰写一篇计算机科学与人工智能领域的学术论文，主题为“基于大语言模型的7-Agent流水线式论文学术写作系统：框架设计与实现”。

## 论文核心内容要求

### 1. 研究背景与问题定义
- 学术论文写作是科研人员的核心任务，但耗时巨大，涉及文献检索、文献分析、格式规范、语言表达等多个环节
- 现有大语言模型（LLM）如GPT、Claude、DeepSeek等在文本生成方面表现优异，但单次对话难以完成复杂的长篇学术写作任务
- 核心问题：如何将LLM的能力系统化地组织为多Agent协作流水线，实现从用户需求到完整学术论文的端到端自动化？

### 2. 系统架构设计（核心章节）
必须详细论述以下设计：

**2.1 整体架构：线性流水线（Linear Pipeline）模式**
- 7个Agent按固定顺序执行，上游输出作为下游输入
- 每个Agent职责单一、接口明确，符合单一职责原则
- 与Map-Reduce、Debate、Hierarchical等其他多Agent模式的对比分析

**2.2 Agent角色定义与分工**
- Agent 1 (论文分类器): 基于LLM的语义分类，将用户需求映射到预定义的论文类型（理科论文/文科论文/工科论文/实验报告/调研报告）
- Agent 2 (RAG格式模板检索): 检索增强生成（RAG）技术的应用——使用Sentence-Transformers进行文本向量化，通过余弦相似度在向量数据库中检索最匹配的学术格式模板
- Agent 3 (关键词提取器): LLM驱动的三层关键词提取（核心/次要/边缘），并自动构建学术数据库检索查询式
- Agent 4 (文献检索器): 多源文献检索与三级回退容错链设计——arXiv公开API → 网页搜索引擎 → 模拟数据，每级设有触发阈值
- Agent 5 (文献分析器): 基于LLM的文献综合分析方法——识别研究空白（Research Gap）、提取核心共识、构建创新方向提案
- Agent 6 (论文撰写器): 一次性整体生成策略（vs. 分章节生成），以及中英双语输出版本的设计原理
- Agent 7 (格式校验器): 规则引擎 + LLM审查的双重校验机制

**2.3 关键技术机制**
- 统一LLM接口与依赖倒置：所有Agent共享同一个LLM客户端，通过System Prompt实现角色分化，Agent代码与具体LLM实现解耦
- 字数自适应控制：从用户需求中自动解析目标字数，按四级（short/medium/long/ultra）动态调整文献收集量、检索深度等参数
- 结构化数据流：Agent间通过强类型字典传递数据，避免原始文本传递导致的信息衰减
- 容错与回退机制：每个Agent均有fallback策略，保证系统在任何条件下都能产出可用结果

### 3. 系统实现
- 编程语言与框架：Python
- LLM后端：DeepSeek API（deepseek-chat模型）
- 向量化模型：all-MiniLM-L6-v2（Sentence-Transformers）
- 文献数据源：arXiv Public API、Web Search
- 输出格式：Markdown + JSON元数据
